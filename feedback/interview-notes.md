# User Interview — Documentation Assistant

- **Who:** Classmate (fellow developer working on similar projects)
- **When:** July 6, 2026
- **How:** In person after class — ~15 min

## What they do today (without your project)

When they need to understand unfamiliar code, they usually read through it line by line and add comments manually. For documentation, they copy code into ChatGPT and re-type the same prompt every time: "Explain this code and add comments." It works but gets repetitive, and the responses aren't always consistent.

## What they liked

- The task tabs make it clear what each option does — no guessing
- Having all five tasks in one place instead of writing different prompts each time
- The code editor feels clean and simple, not overwhelming
- Fast responses — didn't have to wait long

## What confused them / what's missing

- Couldn't find a way to upload a file directly — had to copy and paste the code
- Didn't realize at first that they needed to select a language before generating
- Wanted to see the original code and the documented version side by side
- No way to save or export the results — had to copy manually

## What would make them actually use it

- File upload support so they can drag in a `.js` or `.py` file directly
- Export as PDF or Markdown download for easier sharing and access
- Markdown preview for generated content to see formatted output before copying
- History feature to see past analyses (though they understand this needs a database)
- Better language selector UI with real language logos (JS, Python, etc.) instead of just text
- Support for more programming languages beyond the current 6

## What I'll change (next steps)

- [ ] Add file upload or drag-and-drop support
- [ ] Add markdown preview for generated documentation content
- [ ] Add export/download as PDF for easier access and sharing
- [ ] Improve language selector: use real language SVG icons + names instead of plain text
- [ ] Add support for more programming languages
